## CLI Commands

The `neutrond` command-line interface allows users to interact with the Dex module using **positional arguments**.

### Transactions

#### Deposit

Add liquidity to one or more pools.

```bash
neutrond tx dex deposit [receiver] [token-a] [token-b] [amounts-a] [amounts-b] [tick-indexes] [fees] [disable-autoswap] [fail-tx-on-bel] [flags]
```

**Parameters:**
- `receiver`: Address to receive the liquidity position
- `token-a`: First token in the pair
- `token-b`: Second token in the pair  
- `amounts-a`: Comma-separated amounts of token A to deposit
- `amounts-b`: Comma-separated amounts of token B to deposit
- `tick-indexes`: Comma-separated tick indexes (use brackets for negative values: `[-10,5]`)
- `fees`: Comma-separated fee tiers in basis points
- `disable-autoswap`: Comma-separated boolean values for autoswap options
- `fail-tx-on-bel`: Comma-separated boolean values for BEL (Below Expected Liquidity) failure options

**Flags:**
- `--swap-on-deposit`: Enable swap on deposit (optional)

**Examples:**

Single position:
```bash
neutrond tx dex deposit \
  alice \
  untrn \
  ibc/27394FB092D2ECCD56123C74F36E4C1F926001CEADA9CA97EA622B25F41E5EB2 \
  1000000 \
  1000000 \
  0 \
  3000 \
  false \
  false \
  --from mykey \
  --chain-id neutron-1
```

Multiple positions:
```bash
neutrond tx dex deposit \
  alice \
  untrn \
  ibc/27394FB092D2ECCD56123C74F36E4C1F926001CEADA9CA97EA622B25F41E5EB2 \
  1000000,1000000,1000000 \
  1000000,1000000,1000000 \
  [-100,0,100] \
  3000,3000,3000 \
  false,false,false \
  false,false,false \
  --from mykey \
  --chain-id neutron-1
```

#### Withdrawal

Remove liquidity from one or more pools.

```bash
neutrond tx dex withdrawal [receiver] [token-a] [token-b] [shares-to-remove] [tick-indexes] [fees]
```

**Parameters:**
- `receiver`: Address to receive the withdrawn tokens
- `token-a`: First token in the pair
- `token-b`: Second token in the pair
- `shares-to-remove`: Comma-separated amounts of share tokens to burn
- `tick-indexes`: Comma-separated tick indexes (use brackets for negative values: `[-10,5]`)
- `fees`: Comma-separated fee tiers in basis points

**Example:**
```bash
neutrond tx dex withdrawal \
  alice \
  untrn \
  ibc/27394FB092D2ECCD56123C74F36E4C1F926001CEADA9CA97EA622B25F41E5EB2 \
  500000,250000 \
  [-10,5] \
  3000,3000 \
  --from mykey \
  --chain-id neutron-1
```

#### Place Limit Order

Place a limit order in the DEX.

```bash
neutrond tx dex place-limit-order [receiver] [token-in] [token-out] [tick-index] [amount-in] [order-type] [expiration-time] [flags]
```

**Parameters:**
- `receiver`: Address to receive the order proceeds
- `token-in`: Token being sold
- `token-out`: Token being bought
- `tick-index`: Tick index for the order (use brackets for negative values: `[-10]`)
- `amount-in`: Amount of input token to sell
- `order-type`: Order type (optional, defaults to `GOOD_TIL_CANCELLED`)
  - `GOOD_TIL_CANCELLED` - Remains active until manually cancelled
  - `FILL_OR_KILL` - Must fill completely or fail
  - `IMMEDIATE_OR_CANCEL` - Fill what's possible, cancel the rest
  - `JUST_IN_TIME` - Active for exactly one block
  - `GOOD_TIL_TIME` - Active until expiration time
- `expiration-time`: Expiration time for `GOOD_TIL_TIME` orders (format: `MM/dd/yyyy HH:mm:ss`)

**Flags:**
- `--max-amount-out`: Maximum amount of output token to receive
- `--price`: Limit price for the order

**Examples:**

Basic limit order:
```bash
neutrond tx dex place-limit-order \
  alice \
  untrn \
  ibc/27394FB092D2ECCD56123C74F36E4C1F926001CEADA9CA97EA622B25F41E5EB2 \
  [-10] \
  1000000 \
  GOOD_TIL_CANCELLED \
  --from mykey \
  --chain-id neutron-1
```

Time-limited order:
```bash
neutrond tx dex place-limit-order \
  alice \
  untrn \
  ibc/27394FB092D2ECCD56123C74F36E4C1F926001CEADA9CA97EA622B25F41E5EB2 \
  [-10] \
  1000000 \
  GOOD_TIL_TIME \
  "12/31/2024 23:59:59" \
  --max-amount-out 95000 \
  --price 0.095 \
  --from mykey \
  --chain-id neutron-1
```

#### Multi-Hop Swap

Execute a market swap through multiple pools.

```bash
neutrond tx dex multi-hop-swap [receiver] [routes] [amount-in] [exit-limit-price] [pick-best-route]
```

**Parameters:**
- `receiver`: Address to receive the swap output
- `routes`: Semicolon-separated routes, where each route is comma-separated token denominations
- `amount-in`: Amount of input token to swap
- `exit-limit-price`: Minimum acceptable output price
- `pick-best-route`: Boolean flag to choose the best route among those provided

**Examples:**

Single route:
```bash
neutrond tx dex multi-hop-swap \
  alice \
  "untrn,ibc/27394FB092D2ECCD56123C74F36E4C1F926001CEADA9CA97EA622B25F41E5EB2" \
  1000000 \
  0.095 \
  false \
  --from mykey \
  --chain-id neutron-1
```

Multiple routes with best route selection:
```bash
neutrond tx dex multi-hop-swap \
  alice \
  "untrn,ibc/27394FB092D2ECCD56123C74F36E4C1F926001CEADA9CA97EA622B25F41E5EB2;untrn,uusdc,ibc/27394FB092D2ECCD56123C74F36E4C1F926001CEADA9CA97EA622B25F41E5EB2" \
  1000000 \
  0.095 \
  true \
  --from mykey \
  --chain-id neutron-1
```

### Queries

#### Params

The `params` command queries the current Dex module parameters.

```bash
neutrond query dex params
```

Example output:
```json
{
  "params": {
    "max_pool_spread_bps": "100",
    "min_tick_spacing": "1"
  }
}
```

#### Show Pool

The `show-pool` command queries a pool by its parameters.

```bash
neutrond query dex show-pool [pair-id] [tick-index] [fee] [flags]
```

**Note:** Make sure to wrap your pair-id in quotes to prevent shell interpretation of `&lt;&gt;` characters.

Example:
```bash
neutrond query dex show-pool 'untrn&lt;&gt;uatom' 0 3000
```

#### Show Pool By ID

The `show-pool-by-id` command queries a pool by its ID.

```bash
neutrond query dex show-pool-by-id [pool-id] [flags]
```

Example:
```bash
neutrond query dex show-pool-by-id 1
```

#### Show Pool Metadata

The `show-pool-metadata` command queries a pool's metadata.

```bash
neutrond query dex show-pool-metadata [id] [flags]
```

Example:
```bash
neutrond query dex show-pool-metadata 1
```

#### List Pool Metadata

The `list-pool-metadata` command queries all pool metadata.

```bash
neutrond query dex list-pool-metadata [flags]
```

### Simulation Queries

The Dex module provides several simulation queries that allow users to estimate the outcome of operations without executing them:

#### Simulate Deposit

Simulate the outcome of a deposit transaction.

```bash
neutrond query dex simulate-deposit [receiver] [token-a] [token-b] [amounts-a] [amounts-b] [tick-indexes] [fees] [options]
```

#### Simulate Withdrawal

Simulate the outcome of a withdrawal transaction.

```bash
neutrond query dex simulate-withdrawal [receiver] [token-a] [token-b] [shares-to-remove] [tick-indexes] [fees]
```

#### Simulate Place Limit Order

Simulate the outcome of placing a limit order.

```bash
neutrond query dex simulate-place-limit-order [receiver] [token-in] [token-out] [tick-index] [amount-in] [order-type] [expiration-time]
```

#### Simulate Withdraw Filled Limit Order

Simulate the outcome of withdrawing from a filled limit order.

```bash
neutrond query dex simulate-withdraw-filled-limit-order [tranche-key]
```

#### Simulate Cancel Limit Order

Simulate the outcome of cancelling a limit order.

```bash
neutrond query dex simulate-cancel-limit-order [tranche-key]
```

#### Simulate Multi-Hop Swap

Simulate the outcome of a multi-hop swap.

```bash
neutrond query dex simulate-multi-hop-swap [receiver] [routes] [amount-in] [exit-limit-price] [pick-best-route]
```

## REST Endpoints

### Query Endpoints

- `GET /neutron/dex/params` - Query module parameters
- `GET /neutron/dex/limit_order_tranche_user/{address}/{tranche_key}` - Get limit order tranche user
- `GET /neutron/dex/limit_order_tranche_user` - Get all limit order tranche users
- `GET /neutron/dex/limit_order_tranche/{pairID}/{tokenIn}/{tickIndex}/{trancheKey}` - Get limit order tranche
- `GET /neutron/dex/limit_order_tranche/{pairID}/{tokenIn}` - Get all limit order tranches for pair
- `GET /neutron/dex/user/deposits/{address}` - Get user deposits
- `GET /neutron/dex/user/limit_orders/{address}` - Get user limit orders
- `GET /neutron/dex/tick_liquidity/{pairID}/{tokenIn}` - Get tick liquidity
