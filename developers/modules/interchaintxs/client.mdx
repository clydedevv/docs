This document details how to interact with the Interchain Transactions module. Unlike other Neutron modules, the Interchain Transactions module is designed exclusively for CosmWasm smart contracts and cannot be used directly through CLI commands or individual user transactions.

<Warning>
The Interchain Transactions module does not support direct CLI transactions. All interactions must be implemented within CosmWasm smart contracts. Users cannot register interchain accounts or submit transactions directly.
</Warning>

## Contract Integration

The Interchain Transactions module provides CosmWasm contracts with the ability to:

- Register interchain accounts on remote chains
- Submit transactions to be executed on remote chains
- Receive callbacks from IBC acknowledgments and timeouts

### Message Types for Contracts

Contracts can send the following messages to the module:

#### RegisterInterchainAccount

```rust
use neutron_sdk::bindings::msg::NeutronMsg;

let register_msg = NeutronMsg::RegisterInterchainAccount {
    connection_id: "connection-0".to_string(),
    interchain_account_id: "my-account".to_string(),
    register_fee: vec![Coin {
        denom: "untrn".to_string(),
        amount: Uint128::from(1000u128),
    }],
};
```

#### SubmitTx

```rust
let submit_msg = NeutronMsg::SubmitTx {
    connection_id: "connection-0".to_string(),
    interchain_account_id: "my-account".to_string(),
    msgs: vec![/* your cosmos messages */],
    memo: "".to_string(),
    timeout: 3600,
    fee: Fee {
        recv_fee: vec![Coin {
            denom: "untrn".to_string(),
            amount: Uint128::from(2000u128),
        }],
        ack_fee: vec![Coin {
            denom: "untrn".to_string(),
            amount: Uint128::from(1000u128),
        }],
        timeout_fee: vec![Coin {
            denom: "untrn".to_string(),
            amount: Uint128::from(1000u128),
        }],
    },
};
```

## Query Endpoints

While transaction submission is restricted to contracts, query endpoints can be accessed through CLI for debugging and monitoring purposes.

### Query Module Parameters

```bash
neutron query interchaintxs params
```

Example Response:
```json
{
  "params": {
    "host_enabled": true,
    "allow_messages": [
      "/cosmos.bank.v1beta1.MsgSend",
      "/cosmos.staking.v1beta1.MsgDelegate",
      "/cosmos.staking.v1beta1.MsgUndelegate",
      "/cosmos.staking.v1beta1.MsgBeginRedelegate",
      "/cosmos.distribution.v1beta1.MsgWithdrawDelegatorReward",
      "/cosmos.gov.v1beta1.MsgVote"
    ]
  }
}
```

### Query Interchain Account Address

```bash
neutron query interchaintxs interchain-account-address \
  <owner-address> \
  <interchain-account-id> \
  <connection-id>
```

Example:
```bash
neutron query interchaintxs interchain-account-address \
  neutron1abcdef... \
  myaccount \
  connection-0
```

Example Response:
```json
{
  "interchain_account_address": "cosmos1xyz..."
}
```

## gRPC Queries

The Interchain Transactions module exposes gRPC endpoints for querying state.

### Params

Query the current parameters of the module.

```bash
grpcurl -plaintext \
    localhost:9090 \
    neutron.interchaintxs.v1.Query/Params
```

Example Response:
```json
{
  "params": {
    "hostEnabled": true,
    "allowMessages": [
      "/cosmos.bank.v1beta1.MsgSend",
      "/cosmos.staking.v1beta1.MsgDelegate",
      "/cosmos.staking.v1beta1.MsgUndelegate",
      "/cosmos.staking.v1beta1.MsgBeginRedelegate",
      "/cosmos.distribution.v1beta1.MsgWithdrawDelegatorReward",
      "/cosmos.gov.v1beta1.MsgVote"
    ]
  }
}
```

### InterchainAccountAddress

Query the address of an interchain account on a remote chain.

```bash
grpcurl -plaintext \
    -d '{"owner_address": "neutron1...", "interchain_account_id": "myaccount", "connection_id": "connection-0"}' \
    localhost:9090 \
    neutron.interchaintxs.v1.Query/InterchainAccountAddress
```

Example Response:
```json
{
  "interchainAccountAddress": "cosmos1xyz..."
}
```
