This document details how to interact with the Revenue module using the CLI and gRPC.

## CLI Commands

The Revenue module provides several commands to query state and send transactions.

### Queries

#### Query Module Parameters

```bash
neutrond query revenue params [flags]
```

Example:
```bash
neutrond query revenue params --output json
```

#### Query Validator Stats

```bash
neutrond query revenue validator-stats [validator-addr] [flags]
```

Example:
```bash
neutrond query revenue validator-stats neutronvaloper1... --output json
```

#### Query Payment Info

```bash
neutrond query revenue payment-info [flags]
```

Example:
```bash
neutrond query revenue payment-info --output json
```

#### Query All Validators Stats

```bash
neutrond query revenue validators-stats [flags]
```

Example:
```bash
neutrond query revenue validators-stats --output json
```

### Transactions

#### Fund Treasury

```bash
neutrond tx revenue fund-treasury [amount] [flags]
```

Example:
```bash
neutrond tx revenue fund-treasury 1000000untrn \
  --from mykey \
  --chain-id neutron-1 \
  --gas auto \
  --gas-adjustment 1.4
```

## gRPC Queries

The Revenue module exposes gRPC endpoints for querying state.

### Params

Query the current parameters of the module.

```bash
grpcurl -plaintext \
    localhost:9090 \
    neutron.revenue.Query/Params
```

Example Response:
```json
{
  "params": {
    "baseCompensation": "2000",
    "denomCompensation": "untrn",
    "twapWindowSize": "30",
    "performanceRequirements": {
      "requiredAtLeast": "0.800000000000000000",
      "allowedToMiss": "0.050000000000000000"
    },
    "paymentScheduleType": "PAYMENT_SCHEDULE_TYPE_MONTHLY"
  }
}
```

### ValidatorStats

Query the performance metrics for a specific validator.

```bash
grpcurl -plaintext \
    -d '{"val_oper_address": "neutronvaloper1..."}' \
    localhost:9090 \
    neutron.revenue.Query/ValidatorStats
```

Example Response:
```json
{
  "stats": {
    "validator_info": {
      "address": "neutronvaloper1...",
      "signed_blocks": "980",
      "blocks_in_window": "1000",
      "provided_oracle_votes": "950",
      "oracle_votes_in_window": "1000"
    },
    "total_produced_blocks_in_period": "1000",
    "performance_rating": "0.975000000000000000",
    "expected_revenue": {
      "denom": "untrn",
      "amount": "1950"
    }
  }
}
```

### PaymentInfo

Query the current payment info including schedule and revenue details.

```bash
grpcurl -plaintext \
    localhost:9090 \
    neutron.revenue.Query/PaymentInfo
```

Example Response:
```json
{
  "payment_schedule": {
    "type": "PAYMENT_SCHEDULE_TYPE_MONTHLY",
    "periods_passed": "5",
    "periods_count": "0",
    "month_end_time": "2023-11-01T00:00:00Z"
  },
  "effective_period_progress": "0.750000000000000000",
  "reward_asset_twap": "0.095000000000000000",
  "base_revenue_amount": {
    "denom": "untrn",
    "amount": "2000"
  }
}
```

### ValidatorsStats

Query all validators' performance metrics.

```bash
grpcurl -plaintext \
    localhost:9090 \
    neutron.revenue.Query/ValidatorsStats
```

## gRPC Transactions

The Revenue module supports the following transactions via gRPC.

### FundTreasury

Fund the revenue treasury with tokens.

```bash
grpcurl -plaintext \
    -d '{
      "sender": "neutron1...",
      "amount": [
        {
          "denom": "untrn",
          "amount": "1000000"
        }
      ]
    }' \
    localhost:9090 \
    neutron.revenue.Msg/FundTreasury
```

## REST API

The Revenue module exposes REST endpoints for querying state. The base URL is typically `http://localhost:1317`.

### Query Parameters

```
GET /neutron/revenue/params
```

Example Response:
```json
{
  "params": {
    "base_compensation": "2000",
    "denom_compensation": "untrn",
    "twap_window_size": 30,
    "performance_requirements": {
      "required_at_least": "0.800000000000000000",
      "allowed_to_miss": "0.050000000000000000"
    },
    "payment_schedule_type": "PAYMENT_SCHEDULE_TYPE_MONTHLY"
  }
}
```

### Query Validator Stats

```
GET /neutron/revenue/validator_stats/{validatorAddr}
```

Example Response:
```json
{
  "stats": {

    "signed_blocks": 980,
    "blocks_in_window": 1000,

}
```
