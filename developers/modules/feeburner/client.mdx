This document describes the client interface for interacting with the Fee Burner module.

<Warning>
**Query-Only Access**: The Fee Burner module only provides query functionality to external clients. Transaction operations (`MsgUpdateParams`) can only be executed through governance proposals, not directly by individual users.
</Warning>

The `neutrond` command-line interface provides several commands to interact with the Fee Burner module.

### Queries

#### Module Parameters

The `params` command queries the current module parameters.

```bash
neutrond query feeburner params
```

Example output:
```json
{
  "params": {
    "neutron_denom": "untrn",
    "reserve_address": "",
    "treasury_address": "neutron1..."
  }
}
```

#### Total Burned NTRN Amount

The `total-burned-neutrons-amount` command queries the total amount of NTRN tokens burned since genesis.

```bash
neutrond query feeburner total-burned-neutrons-amount
```

Example output:
```json
{
  "total_burned_neutrons_amount": {
    "coin": {
      "denom": "untrn",
      "amount": "1532687456789"
    }
  }
}
```

<Note>
**No Transaction Commands**: The Fee Burner module does not provide CLI transaction commands for individual users. Parameter updates must be submitted through governance proposals.
</Note>

## gRPC Usage

The Fee Burner module exposes gRPC endpoints that can be accessed programmatically:

### Querying Module Parameters

```bash
grpcurl -plaintext \
  -d '{}' \
  localhost:9090 \
  neutron.feeburner.Query/Params
```

Example output:
```json
{
  "params": {
    "neutronDenom": "untrn",
    "reserveAddress": "",
    "treasuryAddress": "neutron1..."
  }
}
```

### Querying Total Burned NTRN Amount

```bash
grpcurl -plaintext \
  -d '{}' \
  localhost:9090 \
  neutron.feeburner.Query/TotalBurnedNeutronsAmount
```

Example output:
```json
{
  "totalBurnedNeutronsAmount": {
    "coin": {
      "denom": "untrn",
      "amount": "1532687456789"
    }
  }
}
```

### Governance Message Example

For governance proposals that update parameters, the gRPC message structure would be:

```json
{
  "authority": "neutron1...", // governance authority address
  "params": {
    "neutronDenom": "untrn",
    "treasuryAddress": "neutron1...",
    "reserveAddress": ""
  }
}
```

<Note>
This message can only be executed through governance proposals, not directly by individual users.
</Note>

## JavaScript/TypeScript Client Usage

Example of querying the Fee Burner module using CosmJS:

```typescript
import { QueryClient, createProtobufRpcClient } from "@cosmjs/stargate";
import { Tendermint34Client } from "@cosmjs/tendermint-rpc";
import { Query } from "neutron-typegen/neutron/feeburner/query";

// Connect to a Tendermint RPC endpoint
const tendermintClient = await Tendermint34Client.connect("http://localhost:26657");
const queryClient = QueryClient.withExtensions(tendermintClient);

// Create a Fee Burner module query client
const rpcClient = createProtobufRpcClient(queryClient);
const feeburnerQueryClient = {
  params: () => {
    return Query.params({});
  },
  totalBurnedNeutronsAmount: () => {
    return Query.totalBurnedNeutronsAmount({});
  },
};

// Query module parameters
const params = await feeburnerQueryClient.params();
console.log("Fee Burner Parameters:", params);

// Query total burned NTRN amount
const burnedAmount = await feeburnerQueryClient.totalBurnedNeutronsAmount();
console.log("Total Burned NTRN:", burnedAmount.totalBurnedNeutronsAmount.coin);
```

## Governance Integration

For developers working on governance proposals that need to update Fee Burner parameters:

```javascript
// Example governance proposal message
const updateParamsMsg = {
  "@type": "/neutron.feeburner.MsgUpdateParams",
  "authority": "neutron1...", // governance authority
  "params": {
    "neutron_denom": "untrn",
    "treasury_address": "neutron1...",
    "reserve_address": ""
  }
};

// This would be included in a governance proposal
const proposal = {
  "title": "Update Fee Burner Parameters",
  "description": "Update treasury address for non-NTRN fees",
  "messages": [updateParamsMsg]
};
```

This client interface ensures that users can query Fee Burner module state while parameter updates remain under democratic governance control.
